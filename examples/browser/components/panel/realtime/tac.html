How durable delivery works
  • Registration stores the client in a Yon-managed list and returns a durable clientId.
  • Connecting opens a one-way Server-Sent Events stream at /_yon/realtime/stream.
  • Messages are written to a LocalQueue mailbox and replayed from a persisted cursor on reconnect.
  • This is durable one-way push over SSE, not a true bidirectional WebSocket.
